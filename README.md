# 💻 Projeto ADS – Sistema Integrado

---

## 📌 Sobre o Projeto

O **Projeto ADS Sistema Integrado** é uma iniciativa acadêmica voltada ao desenvolvimento de uma aplicação robusta e completa. O sistema foi planejado para integrar diferentes camadas de tecnologia, garantindo uma experiência fluida desde a persistência de dados até a interface com o usuário final.

O projeto foca na integração de:
* **Banco de Dados:** Estruturação e modelagem de dados.
* **Lógica de Processamento:** Desenvolvimento do back-end e regras de negócio.
* **Interface Web:** Criação de um front-end responsivo e intuitivo.

---

## ⚙️ Guia de Contribuição e Versionamento

Para garantir a integridade do código e a organização do projeto, todos os desenvolvedores devem seguir estritamente os padrões abaixo:

### 🛑 Regras de Ouro
1. **Nunca** faça commits ou alterações diretamente na branch `main`.
2. É **obrigatório** criar uma branch específica para cada nova funcionalidade.
3. Toda alteração deve ser enviada via **Pull Request (PR)** e aprovada pelo **Grupo 1**.

### 🌿 Padrão de Branches
Utilize os nomes das branches de acordo com o seu grupo ou o tipo de tarefa:
* `grupo2-modelagem`
* `grupo3-logica`
* `grupo4-frontend`
* `feature/nome-da-funcionalidade`

### 💬 Padrão de Commits
Mantenha o histórico de mensagens limpo e padronizado:
* `feat:` para novas funcionalidades.
* `fix:` para correções de erros.
* `docs:` para alterações em documentações.
* `style:` para ajustes visuais ou de formatação que não alteram a lógica.

## 🔄 Fluxo de Trabalho Acadêmico

1. **Clonar** o repositório localmente.
2. Realizar as alterações necessárias.
3. Adicionar os arquivos: `git add .`
4. Realizar o commit: `git commit -m "tipo: descrição clara"`
5. Enviar para o servidor: `git push origin [nome-da-sua-branch]`
6. Abrir o **Pull Request** no GitHub para revisão do Grupo 1.
